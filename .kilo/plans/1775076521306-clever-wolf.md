# Plan: 2-PASS Attention — Full Design (FINAL)

## Müdahale Noktası

**3'lü kontrol noktası:**
```
1. vec_dot_KQ()        → Q·K hesaplanır (K split-aware)
2. softmax kernel      → normalize edilir (TEK softmax)
3. vec_dot_V()         → attention output üretilir (V split-aware)
```

Sadece logits değil — **logits → weight → V çarpımı** da split-aware olmalı.

Write path'i nasıl tasarladığın önemli değil — **attention kernel hâlâ tek buffer varmış gibi davranıyor**. K_high hiç kullanılmaz, yanlış stride ile okunur, veya undefined memory access olur.

## Doğru Müdahale Noktası

### 🔥 `fattn-vec.cuh: vec_dot_KQ()`

Mevcut:
```cpp
for (int i = 0; i < n_kv; ++i) {
    score[i] = dot(Q, K[i])  // tek buffer, tek type
}
```

Hedef:
```cpp
if (token_pos >= split_point) {
    score[i] = dot_turbo4(Q, K_high[i])  // HIGH buffer
} else {
    score[i] = dot_turbo3(Q, K_low[i])   // LOW buffer
}
```

**Ama branch warp divergence yaratır.** Doğru yaklaşım: **2-PASS STRATEJİ**.

## 2-PASS STRATEJİ (TAM TASARIM)

### Neden 2-PASS DOĞRU?

- Her kernel homogeneous kalır (tek type) → warp divergence yok
- Mevcut optimized kernels bozulmaz
- Sadece orchestration değişir
- Branch YOK (KQ ve V tarafında)

### Neden Branch YANLIŞ?

```cpp
// YANLIŞ — warp divergence
if (token_pos >= split_point)
    vec_dot_turbo4(...)
else
    vec_dot_turbo3(...)
```
→ Her warp içinde thread'ler farklı dallara girer → performans ölümü

### Tam 2-PASS Akış (K + V)

```
=== KQ SIDE ===
PASS 1 (LOW - turbo3):
  logits_low = Q · K_low              [n_low tokens]

PASS 2 (HIGH - turbo4):
  logits_high = Q · K_high            [n_high tokens]

=== MERGE (TEK) ===
logits = [logits_low | logits_high]   [n_total tokens]

=== SOFTMAX (TEK — ASLA AYRI DEĞİL) ===
weights = softmax(logits)             [n_total weights]

=== V SIDE (AYRILMAMIŞ SADECE KQ, V DA SPLIT!) ===
PASS 1 (LOW - turbo3):
  out_low  = Σ weights_low[i] * V_low[i]     [n_low tokens]

PASS 2 (HIGH - turbo4):
  out_high = Σ weights_high[i] * V_high[i]   [n_high tokens]

=== FINAL ===
output = out_low + out_high           [D vector]
```

### ⚠️ EN KRİTİK BUG ADAYLARI

**ASLA ŞUNU YAPMA:**
```cpp
softmax(logits_low)   // YANLIŞ!
softmax(logits_high)  // YANLIŞ!
```
→ Model tamamen çöker

**ASLA ŞUNU YAPMA (V tarafı):**
```cpp
out = softmax_low * V_low + softmax_high * V_high  // YANLIŞ!
```
→ Ayrı softmax → normalization bozuk → output yanlış

**DOĞRU:**
```cpp
merge(logits_low, logits_high)  // birleştir
softmax(merged_logits)           // TEK softmax
weights = [weights_low | weights_high]
out_low  = Σ weights_low * V_low
out_high = Σ weights_high * V_high
output = out_low + out_high
```

### Performans Gerçeği

2-pass demek:
- 2x KQ
- 2x V
- 1x merge
- 1x softmax

→ ~2.2x compute cost
Ama:
- VRAM ciddi düşer
- Reasoning korunur
- 128K context mümkün olur

Trade-off net.

### Kernel Seviyesinde Tam Müdahale Haritası

**`fattn.cu` — orchestration:**
```cpp
// --- KQ side ---
launch_KQ_turbo3(Q, K_low, n_low, logits_low);
launch_KQ_turbo4(Q, K_high, n_high, logits_high);

// --- merge logits ---
merge_logits(logits_low, logits_high, logits_total, n_low, n_high);

// --- softmax (TEK) ---
softmax(logits_total, n_total, weights);

// --- V side (split-aware) ---
launch_V_turbo3(weights_low, V_low, n_low, out_low);
launch_V_turbo4(weights_high, V_high, n_high, out_high);

// --- final sum ---
combine_outputs(out_low, out_high, output);
```

**`fattn-vec.cuh` — kernel'ları reuse et:**
- `vec_dot_KQ_turbo3` → PASS 1 KQ
- `vec_dot_KQ_turbo4` → PASS 2 KQ
- `dequantize_V_turbo3` → PASS 1 V
- `dequantize_V_turbo4` → PASS 2 V

Değiştirme, **reuse et**.

## Dikkat Edilecekler

- READ PATH asıl müdahale noktası — write path değil
- V SIDE da split-aware olmalı — sadece K yetmez
- 2-PASS stratejisi → warp divergence yok (branch yok)
- Merge BEFORE softmax — ASLA ayrı softmax
- V output = out_low + out_high (split-aware weighted sum)
- **Logits normalization before merge** — turbo3 vs turbo4 farklı scale distribution. Normalization yoksa turbo4 dominate eder
- **Weights slicing doğru offset** — stride farklı, head packing farklı → silent corruption
- Lazy batch transfer ilk prototipte KAPALI
- Attention reweighting ilk prototipte KAPALI
- **Real cost: 2.3-2.6x** (2x memory + 2x launch + cache locality kaybı)
- Değiştirme, reuse et — mevcut kernel'ları tekrar kullan
- llama.cpp fused pipeline'ı bölüyorsun — pipeline uyumsuzluğu riski

## ⚠️ Logits Normalization (KRİTİK)

**Sorun:** turbo3 (low variance) ve turbo4 (high precision) merge sonrası turbo4 dominate eder.

// Softmax ÖNCESİ normalization
